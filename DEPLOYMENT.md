# スマートフォンで使う：Vercel デプロイ手順

このアプリを**携帯のホーム画面に追加して、アプリのように使う**方法を詳しく説明します。  
難しい設定はなく、**クリックだけ**で5分で完了します。

---

## 🎯 全体の流れ（3ステップ）

```
1. Vercel で GitHub と繋ぐ        (3分)
2. アプリをデプロイする          (3分、自動)
3. スマートフォンでホーム画面に追加  (1分)
```

---

## ステップ 1：Vercel にサインアップ

### 1-1. Vercel のサイトを開く

ブラウザで以下を開いてください：
```
https://vercel.com
```

トップページが見えると思います。右上に **「Sign Up」** というボタンがあります。

### 1-2. GitHub でログイン

**「Sign Up」** をクリックすると、ログイン方法を選ぶ画面が出ます。

```
複数の選択肢が見えます：
  ☐ Google でサインアップ
  ☐ GitHub でサインアップ    ← これをクリック
  ☐ GitLab でサインアップ
```

**「GitHub でサインアップ」** をクリックしてください。

### 1-3. GitHub の認可画面

Vercel から「GitHub にアクセスしていい？」という確認が出ます。

画面に以下のような文が見えます：
```
Vercel は以下の権限を求めています：
  ✓ リポジトリへのアクセス
  ✓ デプロイ設定の読み取り
```

**「認可する」** や **「Authorize」** ボタンをクリックしてください。

### 1-4. 登録完了

Vercel のダッシュボード（管理画面）が開きます。

```
画面に見えるもの：
  - 左上：Vercel のロゴ
  - 画面中央：「Add New」ボタン
  - 右上：プロフィール
```

これでステップ 1 完了 ✅

---

## ステップ 2：GitHub リポジトリをインポート

### 2-1. 「Add New」をクリック

Vercel のダッシュボードで、画面中央の **「Add New」** ボタンをクリックしてください。

メニューが出ます：
```
  ├─ Project        ← これをクリック
  ├─ Environment Variable
  ├─ Monitoring
  └─ Team
```

**「Project」** をクリック。

### 2-2. リポジトリを選ぶ

新しいページが開きます。タイトルは：
```
「Import Git Repository」
```

ページ上部に**検索ボックス**があります。その下に、あなたの GitHub リポジトリが列挙されているはずです。

スクロールして以下を探してください：
```
Nyasushi825/my-app
```

もし見当たらなければ、検索ボックスに `my-app` と入力して検索してください。

**「Nyasushi825/my-app」** が見つかったら、その行をクリック。

### 2-3. ブランチを指定（重要）

リポジトリを選ぶと、次のページが開きます。タイトルは：
```
「Configure Project」
```

このページをスクロール下げると、以下の項目が見えます：

```
Production Branch
  ┌─────────────────┐
  │ main      ▼     │  ← ここをクリック
  └─────────────────┘
```

**「Production Branch」** の右側にあるドロップダウン（▼）をクリック。

リストが出ます：
```
  ├─ main                          
  ├─ claude/trusting-allen-Xk9Oi   ← これを選ぶ
```

**「claude/trusting-allen-Xk9Oi」** をクリック。

### 2-4. デプロイ開始

同じページの下部に **「Deploy」** という大きなボタンがあります。

**「Deploy」** をクリック。

すると、デプロイが始まります。画面に：
```
Deploying...
Building your project (this may take a few seconds)
```

という文が見えます。

**ここから先は何もしないでください。** Vercel が自動的にビルド・デプロイをしてくれます。

### 2-5. デプロイ完了を待つ

1〜3分で完了します。画面が以下のように変わります：

```
✓ Deployment Successful!

Visit
https://my-app-abc123.vercel.app    ← あなたのアプリの URL
```

この URL （`https://...` で始まる）をコピーしておいてください。  
これがあなたのアプリの住所です。

**これでステップ 2 完了 ✅**

---

## ステップ 3：スマートフォンでホーム画面に追加

### 3-1. スマートフォン・タブレットを用意

ここからはスマートフォンまたはタブレットで進めます。

### 3-2. ブラウザで URL を開く

スマートフォンのブラウザ（Safari / Chrome など）を開いて、ステップ 2 でコピーした URL を貼り付けます。

```
https://my-app-abc123.vercel.app
```

**Enter キーを押して、アプリを開きます。**

### 3-3. メニューをタップ

ブラウザのメニューボタンをタップします。場所はブラウザによって異なります：

**iPhone・Safari の場合：**
```
画面下の中央 → 上矢印のボタン（↑） をタップ
```

**Android・Chrome の場合：**
```
画面右上の 3 点（⋮）をタップ
```

メニューが出ます。

### 3-4. 「ホーム画面に追加」をタップ

メニューの中から、以下のいずれかを探してタップ：

```
☐ ホーム画面に追加   （iPhone）
☐ アプリをインストール （Android）
☐ ホーム画面に追加   （汎用）
```

### 3-5. アプリ名を確認

次の画面で、アプリ名とアイコンが見えます：

```
アプリ名：「サブスクBox」
アイコン：青い背景に白い四角のロゴ
```

確認して、**「追加」** または **「ホーム画面に追加」** をタップ。

### 3-6. 完成！ 🎉

スマートフォンのホーム画面にアプリが追加されました。

「サブスクBox」というアイコンをタップすると、**ネイティブアプリのように起動します。**

## よくある質問 Q&A

### Q1: デプロイが失敗した（エラーが出た）

**A:** Vercel のダッシュボードで、「Deployments」タブを開いてください。失敗したデプロイの詳細ログが見えます。  
99% は GitHub 権限がないか、ブランチ指定が間違っています。確認して **「Redeploy」** ボタンを押してもう一度試してください。

### Q2: URL が発行されない（ずっと「Deploying...」）

**A:** 
- ページをリロード（F5）してみてください
- 5 分以上待っても変わらない場合、ブラウザを閉じて 30 秒待ち、もう一度 Vercel を開いてください

### Q3: ホーム画面に追加後、アプリを開くと真っ白

**A:**
- ブラウザを一度閉じて、アプリアイコンから再度開いてください
- キャッシュをクリア：ブラウザの設定 → キャッシュをクリア → アプリを再度開く

### Q4: アプリのデータ（登録したサブスク）は保存される？

**A:** はい。このアプリは **localStorage** という端末内のメモリにデータを保存しているので、登録したサブスクは消えません。  
ただし、端末のブラウザを初期化したり、キャッシュを全削除したら消えます。

### Q5: 複数の端末で同じデータを共有したい

**A:** 現在はできません。ただし今後、データベース連携（Supabase など）を追加すれば可能になります。

### Q6: アプリを更新したい（新機能・バグ修正）

**A:** 
- GitHub の開発ブランチ（`claude/trusting-allen-Xk9Oi`）でコード更新
- GitHub にプッシュ
- Vercel が**自動的に**最新コードをビルド・デプロイ

手動操作は不要です。ホーム画面のアプリを開くと、自動的に最新版が表示されます。
